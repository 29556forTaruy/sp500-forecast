# S&P 500 予測モデル & 日次更新アプリ 開発計画

**改訂: 2026-06-13** — ベースモデル(Level 0)の詳細仕様を確定。依頼者の「PERレンジ・中央値 × EPS・成長率による一次関数ベースモデル」案を検討し、対数空間での定式化として採用(§5.1、付録A参照)。

## 0. プロジェクトの定義

**作るもの:** 多数の経済・市場指標から「今後1年の S&P 500 の値動きの確率分布」を推定するモデルと、それを毎日自動更新して表示するアプリ。

**設計上の大原則(統計的誠実さ):**
- 点予測ではなく分布予測(中央値+分位点レンジ、ファンチャート)を出力する
- 必ずナイーブなベースライン(ランダムウォーク+ドリフト、過去平均リターン)と比較し、勝てない要素は削る
- 約150年のデータがあっても、独立な「1年」は150個しかない。実効サンプル数は極めて少ないという前提で過学習を最大の敵とする
- アプリには「過去の予測と実際の答え合わせログ」を必ず組み込み、モデルの実力を常に可視化する
- **ベース数値は単独の「正解の数字」として見せない。** 構造アンカー(Level 0)は常にベースライン(ランダムウォーク+ドリフト)と並記し、2〜3有効数字に丸めて表示する(アンカリングバイアス・偽精度の防止)

## 2. フェーズ1: 予測ターゲットの確定

最初に決めること(推奨案つき):

- **予測対象:** 1年後のトータルリターン(配当込み)か価格リターンか → 推奨は価格リターン(SPX指数そのもので答え合わせしやすい)+参考値として配当利回り表示。Level 0 の構造分解(Bogle分解: 投資リターン=配当利回り+EPS成長、投機的リターン=PER変化)とも整合する
- **出力形式:** 1年後リターンの分位点(5/25/50/75/95%)+ 月次の経路分布(ファンチャート用)
- **更新頻度:** 予測は毎日再計算、モデル自体の再学習は月次または四半期
- **評価指標:** OOS R²(Campbell–Thompson 流)、方向的中率、分位点のカバレッジ(80%区間に実際80%入るか)、ピンボール損失/CRPS

|---|---|---|
| バリュエーション | CAPE(Shiller PE)、益回り、配当利回り、時価総額/GDP | Shiller 公開データ(1871年〜)、FRED |
| 金利・金融政策 | FF金利、10Y−3M / 10Y−2Y スプレッド、実質金利(10年TIPS)、FRB総資産、M2 | FRED |
| 景気マクロ | 失業率(+Sahmルール)、鉱工業生産、ミシガン消費者信頼感、製造業サーベイ | FRED |
| クレジット・リスク | ハイイールドOAS、VIX、シカゴ連銀金融環境指数(NFCI) | FRED、CBOE |
| 市場内部 | 12-1ヶ月モメンタム、200日線乖離率、AAII ブル・ベア比 | yfinance、AAII |
| 企業収益 | 実績EPS・利益率、予想EPS | Shiller データ、S&P公式のEPS集計Excel |
| テクニカル(要検証) | 年間ピボット(前年の高値・安値・終値から P / R1 / S1 / R2 / S2)、現在値のピボット乖離率 | yfinance の日次データから自前算出 |

注意: ISM PMI と Conference Board LEI は本体データが有料化されており、月次発表値を手動収集するか代替系列(地区連銀サーベイ等)を使う判断が必要。予想EPSの時系列も無料入手は難しいため、S&P公式Excelの範囲で妥協するのが現実的。

### データ収集の技術構成
- **FRED API**(無料、要APIキー登録)がマクロ系の基幹(当面は APIキー不要の fredgraph.csv で代替)
- **yfinance** で SPX・VIX 等の市場データ
- Shiller データは Excel を直接ダウンロード

### 最重要の落とし穴(ここで手を抜くとモデルが嘘をつく)
- **発表ラグ:** GDPや雇用統計は1〜3ヶ月遅れて公表される。「その時点で見えていた値」だけを使う(リアルタイム整合性)
- **データ改定:** マクロ統計は後から改定される。厳密にはポイントインタイムのビンテージデータ(FREDのALFRED)が理想。最低限、発表ラグ分のシフトは必須
- **生存バイアス:** 「効くと有名な指標」は過去に効いたから有名なだけ、という可能性を常に疑う

### Shiller データ固有の注意(検証で確認済み)
- Shiller の月次価格 P は**日次終値の月中平均**(1926年以降。1926年以前は Cowles 指数由来、最新月は単日終値のことがある)。yfinance の月末終値と混ぜるときは系統的オフセットに注意
- Shiller の月次 EPS は四半期報告の**線形補間**で、確報まで1〜2四半期の公表ラグがある。バックテストでは EPS 由来列(E, CAPE, E10)を3ヶ月ラグさせた変種を「正味の性能」として扱う
- `P/CAPE` で復元されるのは「**CPI実質化した10年平均EPSを t 時点ドルで表した値**(E10n)」であり、名目EPSの単純10年平均ではない(乖離は中央値~11%)。Level 0 の EPS にはこの E10n を使う

## 4. フェーズ3: 探索的分析・特徴量設計

- 各指標 vs その後1年リターンの散布図・相関を総当たりで確認
- 変換の標準化: YoY変化率、過去10年ローリング z スコア、ヒストリカル・パーセンタイル
- **重複ウィンドウ問題:** 月次サンプリングした1年リターンは強い自己相関を持つ(誤差は理論上 MA(11)、ナイーブな t 値は約3倍過大)。Hodrick (1992) または Newey–West(ラグ18)補正、非重複(年次)サンプルでの再確認、ブロックブートストラップをルール化
- **Stambaughバイアス:** log CAPE は月次 AR(1) ≈ 0.996 のほぼ単位根で、そのイノベーションはリターンと強く負相関 → 予測力が過大評価される。バイアス補正後に有意性が消えるなら、データ推定値でなく経済的事前分布(λ≈0.1)に頼る
- ここで「ストーリーは美しいが統計的に何も言えない指標」を大量に落とすことになる(それが正常)

### Level 0 用の特徴量アーティファクト(このフェーズで作成)
`data/processed/level0_features_monthly.csv`:
`date, log_p, log_cape, e10n(=P/CAPE), g20_e10n, cape_star_360m, cape_q05_360m, cape_q25_360m, cape_q75_360m, cape_q95_360m, cape_pctile_360m, val_gap(=ln cape_star − ln CAPE), fwd12_log_ret`
- すべてポイントインタイム(t 時点までのデータのみで計算)。未来参照列は `fwd` 接尾辞付きの教師ラベルのみ
- フェーズ2の出力CSVスキーマ(shiller_monthly.csv 等)は変更しない。派生列はこのファイルに分離する

## 5. フェーズ4: モデル構築(段階的に)

| レベル | モデル | 役割 |
|---|---|---|
| 0 | 構造アンカー(詳細は §5.1): ln P̂ = ln P_t + ĝ + λ̂·(ln CAPE* − ln CAPE_t)。PER中央値への部分回帰 × 平滑化EPS成長。バンドはPER分位点を λ 倍に縮めて構成 | 解釈可能な背骨。全予測の基準点。λ・CAPE*・g はすべてポイントインタイム推定 |
| 1 | ランダムウォーク+ドリフト、過去平均 | 倒すべきベースライン |
| 2 | CAPE等の単回帰 → ElasticNet(正則化線形) | 多重共線性の強い指標群の本命。**Level 0 の残差を学習する(残差モード)** |
| 3 | 勾配ブースティング(LightGBM) | 非線形性の検証。過学習に最警戒。残差モード |
| 4 | 分位点回帰 / ベイズ回帰 | 点予測を分布予測に拡張。ベイズ回帰では Level 0 アンカーを事前平均に使う |
| 5 | 期待リターン(上記)+ ボラティリティ(GARCH or VIX 起点)→ モンテカルロで1年パス生成 | ファンチャートの実体。ドリフトは Level 0 アンカー中心 |

最終形は「複数モデルの単純平均アンサンブル」が有力。予測タスクではシンプルな平均が単独モデルに勝ちやすいことが知られている。

**日次補正レイヤー(2層構成):** 月次〜四半期で再学習する「遅い」期待リターンモデルの上に、毎日の終値確定ごとに ①ファンチャートの起点を最新終値へ再アンカー、②直近の予測誤差でドリフト項を微修正する「速い」レイヤーを載せる。実装候補は状態空間モデル(カルマンフィルタ)または残差ベースの簡易調整で、フェーズ4で比較検証する。**注意: 再アンカー後の系列は疑似ランダムウォーク化するため、精度指標は必ず「純粋なアンカー(月次時点の予測)」に対して計算する。再アンカー系列で測ると見かけの精度が捏造される。**

### 5.1 ベースモデル(Level 0)詳細仕様

依頼者の発案「PERのレンジ・中央値と EPS・その成長率から S&P 500 の軸となる概算値を一次関数で表す」を、以下のとおり**対数空間の一次関数**として正式採用する。P = PER × EPS は掛け算の恒等式なので、対数をとってはじめて厳密に線形(アフィン)になる。生の価格水準での一次関数は150年データ(P: 約5 → 6000)では分散不均一・見せかけの回帰・直近データ支配で破綻するため禁止(根拠は付録A)。

**中央値予測(月次、t = Shiller 月次行):**

```
ln P̂(t+12) = ln P_t + ĝ_t + λ̂_t · (ln CAPE*_t − ln CAPE_t)
```

等価な掛け算形(ユーザー向け説明に有用): `P̂ = E10n_t · exp(ĝ_t) · CAPE_t^(1−λ̂) · (CAPE*_t)^λ̂` — 予測PERは「現在のPER」と「歴史的中央値」の幾何内挿。λ=0 で純粋なEPS成長ドリフト、λ=1 で中央値への完全回帰(これが依頼者の原案に相当するが、採用しない — 下記)。

**分位点バンド(依頼者の「PERレンジ」の反映):**

```
ln P̂_q(t+12) = ln P̂(t+12) + κ · λ̂_t · (ln Q_q,t − ln CAPE*_t),  q ∈ {5, 25, 75, 95%}
```

歴史的PERレンジをそのまま価格帯にせず、**回帰係数 λ の分だけ縮めて**適用する。理由: 歴史的PERレンジは「無条件分布」であり、1年先の条件付き分布(現在のPER近傍に中心がある)とは別物。そのまま使うと帯が広すぎ(±30〜40%)かつ中心がずれ、例えば CAPE 35 のとき「1年内の大暴落がほぼ確実」という歴史的に支持されない表示になる。

**記号の定義とパラメータ推定(すべてポイントインタイム):**

| 記号 | 定義 | 推定方法 |
|---|---|---|
| P_t | Shiller 月次価格(日次終値の月中平均) | Shiller P列 |
| CAPE_t | Shiller CAPE(1881年〜) | Shiller CAPE列 |
| E10n_t | P_t / CAPE_t(実質10年平均EPSの t 時点ドル表示) | 導出列 |
| CAPE*_t | **360ヶ月(30年)ローリング中央値**(依頼者の「PER中央値」) | t 以前のデータのみ。全期間中央値は使用禁止(付録A) |
| Q_q,t | 同じ360ヶ月窓の CAPE 分位点(「PERレンジ」) | 同上 |
| ĝ_t | E10n の過去20年の年率対数成長 | t 以前のみ。直近1年成長の外挿は禁止 |
| λ̂_t | 1年回帰速度 | 拡大窓回帰(1911年〜)、毎月再推定、**[0, 0.35] にクリップし 0.10 へ縮小**、Newey–West(18) SE |
| κ | バンド幅校正スカラー | 初期値1、年1回12月に再校正(目標: 5–95%帯の実績カバレッジ90%)、[0.5, 3] にクランプ |

**実測値の目安(Shillerデータでの再現回帰により確認済み、付録A):** λ̂ ≈ 0.07〜0.11(中心 ~0.09、半減期 ~7年)、1年ホライズンの R² は 2〜6%。**完全回帰(λ=1)は実データの約10倍の回帰速度を仮定することになり、例えば1997年1月(CAPE≈28)に約−40%を予測して実際は+31%、という規模の失敗を量産する。**

**日次再アンカー(アプリの「速い」レイヤーとの接続):**
当日終値 C_d に対し `CAPE_d = CAPE_月次 × C_d / P_月次`(月内は E10n 固定なのでPERは価格比例)とし、同じ式で再計算する。感応度は `d ln P̂ / d ln C = 1 − λ < 1`、つまり市場が5%下げてもアンカーは ~(1−λ)·5% しか下がらない自己安定挙動になる(バリュエーションアンカーとして望ましい性質)。g・λ・CAPE* は月次再推定まで凍結。直近誤差によるドリフト微修正は対数で ±2% 以内の加算項に制限し、構造アンカーを上書きしない。

**他レベルとの統合:**
- Level 2–4 は**残差モード**: 教師 = `ln P(t+12) − ln P̂₀(t+12)`。ElasticNet の「ゼロへの縮小」が「構造アンカーへの縮小」を意味するようになる。Level 4 ベイズ回帰ではアンカーを事前平均に使う
- Level 5 モンテカルロはドリフトをアンカー(+Level 2–4 残差補正)に置き、シミュレーション後に全パスへ一様な対数シフトを掛けて12ヶ月中央値をアンカーに一致させる。MCの5–95%幅が構造バンド幅の [0.6, 1.6] 倍を外れたらアラート
- **UI表記:** 構造バンドは「歴史的バリュエーションレンジ」とラベルし、確率区間(Level 4/5 の校正済み分位点)とは明確に区別する。アンカー値は2〜3有効数字に丸め、ランダムウォーク基準値と必ず並記

**ガードレール(レッドチーム検証で特定した失敗モードへの対策):**
1. **素のtrailing PER をシグナルに使わない。** 2008–09年に as-reported EPS が−77.5%($66.18→$14.88、Q4-08 は史上初のマイナス四半期 −$23.25)となり、trailing P/E は2009年5月に記録の~124 へ暴騰 — 「中央値回帰」モデルは世代的大底で−80%級の暴落を予測する。CAPE(E10)のみ使用。比較用に trailing PER 変種を並走ログするのは可(本採用はウォークフォワードで勝った場合のみ)
2. **全期間中央値を使わない。** 全期間(1881〜)中央値は16.6、post-1990 平均は~27。全期間中央値をアンカーにすると1991年以降ほぼ35年間一貫して弱気(その間に指数は約16倍)。逆に post-1990 平均だと1999年12月(CAPE 44)のバブルを「正常の~60%上」程度にしか見ない。30年ローリング中央値はこのトレードオフの中庸で、どちらの選択も「マクロレジームへの賭け」であることを文書化する
3. **gは長期に強く縮小。** 直近EPS成長の外挿は2008年(実際−78%)・2001年(−51%)・2010/2021年(リバウンド+45〜70%)で符号ごと間違える。長期実質E10成長は~1.5〜2%/年(検証済み)。インフレレジーム転換(1970年代型)では20年トレーリング g が長期間ずれる既知の限界があり、ブレークイーブン等によるインフレ調整は Level 2+ で導入
4. **エピソード回帰テストをCIに組み込む:** 1921-08, 1974-12, 1997-01, 1999-12, 2009-03, 2021-01 の各時点で、1年予測が [−35%, +60%] の範囲内に収まること
5. **受け入れテスト:** ①恒等式テスト(|P − CAPE×E10n|/P < 0.5%)②黄金手計算フィクスチャ(任意の1時点を手計算し、対数空間で1e-9一致)③退化パラメータ(λ=0 → P·e^g、λ=1 → E10n·e^g·CAPE*)④リーク検査(時点Tで切断して再実行し、T時点の出力がビット一致)⑤カバレッジゲート(ウォークフォワードで5–95%帯の実績カバレッジが [86%, 94%])

## 6. フェーズ5: 検証

- **ウォークフォワード検証(拡大ウィンドウ):** 例えば1990年から1年ずつ前進し、常に「その時点までのデータのみ」で学習→翌1年を予測。Level 0 のパラメータ(λ, CAPE*, g, κ)も毎時点で再計算する
- 時系列にランダム k-fold は禁止
- **重複窓の統計的補正:** Hodrick (1992) または Newey–West(ラグ≥18)SE を標準とし、非重複年次回帰とブロックブートストラップ(ブロック長~24ヶ月)で点推定を照合
- **ビンテージ規律:** Shiller EPS 由来列は3ヶ月ラグ変種をヘッドライン性能とする。FRED 系列は可能なら ALFRED ビンテージ
- 危機時の挙動を個別チェック: 2000、2008、2020、2022(+ §5.1 のエピソードテスト6時点)
- 分位点のキャリブレーション検証(PITヒストグラム等)
- 結果は pre-1950 / post-1950 を分けて報告し、レジーム感応性を明示する
- 判定基準: ベースラインに OOS で勝てない特徴量・モデルは容赦なく削除。**ただし Level 0 アンカーは例外扱いとする:** 1年ホライズンでバリュエーション項の OOS 改善は数%程度が文献・実測の相場であり、点予測でランダムウォーク+ドリフトに僅差でも、(i) 解釈可能な背骨、(ii) Level 2–4 の事前分布/残差基準、(iii) ファンチャートの中央傾斜という役割で残す。その場合も性能は正直に並記する

## 11. フェーズ3 中間結果: Level 0 の実証(2026-06-13、`phase3_level0.py`)

依頼者の「PER×EPS 一次関数アンカー」案を §5.1 の仕様で実装し、1881–2026 の月次データで**ウォークフォワード(各時点tで t 以前のデータのみ使用)**バックテストした。コードは独立エージェントが phase3_level0 をインポートせず全数値を機械精度で再現し、**リーク無し**を確認済み。出力: `data/processed/level0_features_monthly.csv`(ポイントインタイム特徴量)、`level0_backtest_summary.json`、`level0_backtest.png`。

**最も頑健で重要な発見 — 原案の λ=1 は壊滅的:**
- full_reversion(λ=1、現在のPERを1年で中央値へ完全回帰させる=依頼者の素朴な原案)の **OOS R² = −3.18**(RMSE 0.40、ベースラインの約2倍、系統的に強気すぎ)。CAPEが高騰し続けた1980–2005で最悪(−11.7)
- これは §5.1 が「λ=1 ではなく小さい推定 λ」を要求した理由の実データによる確証。**この一点だけでも、アイデアを正しく実装する価値がある**

**正直なスコア — 「勝つ」が、その中身は地味:**

| 期間 | level0 vs RW+ドリフト | drift_only(λ=0) vs RW+ドリフト | バリュエーション増分(level0 vs drift_only) |
|---|---|---|---|
| 全期間(1921–2025) | **+0.081** | +0.063 | +0.020 |
| 戦後(post-1950) | +0.037 | **+0.050** | **−0.014** |
| post-1960 | +0.023 | +0.040 | −0.017 |

- 全期間 +0.08 は**見かけ倒し**: スキルの **79%** が1921–1950 に由来。この時代は RW+ドリフトの「拡大平均」ベンチマークが観測数僅少で弱く、handicapされている。**正直な数字は戦後 ~+0.04**
- しかもその +0.04 の**大半は EPS成長ドリフト項(g20)**であって、バリュエーション回帰ではない。g20 が市場の正のドリフトを最初から捉える「無条件ドリフト効果」が主因
- **戦後はバリュエーション・タイミングがむしろ足を引っ張る**(増分 −0.014)。in-sample の λ は非有意(NW t=1.40、R²=0.015)、Clark–West でも level0 vs drift_only は t≈2.0 で限界的。**1年ホライズンでの CAPE 増分はゼロと確実には区別できない**
- これは Goyal–Welch (2008)・Campbell–Thompson (2008) と完全に整合: 1年の株式予測可能性は小さく、初期・極端データに集中する

**バンドは未較正(予測どおり):** バリュエーションのみの帯は q05–q95 カバレッジ **18.3%**(目標90%)。帯の半幅 ~0.043 に対し実現残差の標準偏差 ~0.187 と一桁狭い。点予測は使えるが、ファンチャートの不確実性は Level 5(GARCH/VIX)を待つ必要がある — §5.1 の設計判断の確証。

**フェーズ4への含意(結論):**
1. Level 0 は「RW+ドリフトに負けない構造的背骨」としては合格(多くのバリュエーション予測子は OOS で負ける中で、これは負けない)。ただし**1年点予測の精度向上はほぼ EPS成長ドリフトの貢献**であり、CAPE 回帰の 1年寄与は実質ゼロ
2. CAPE の本領は §5.1 付録A の通り**長期(7–10年)**と**分布の歪み・極端バリュエーションの裾**。Level 0 の役割は点予測ではなく(i)背骨、(ii)Level 2–4 の残差基準/事前分布、(iii)λ=1 のような破滅の回避、と再確認された
3. 次の優先: drift 項の頑健化(g のインフレ調整=Level 2)と、分布の較正(Level 5)。CAPE 単体に1年の点予測精度を期待しない

## 付録A: ベースモデル採用判断の根拠(2026-06-13 検証)

依頼者の発案を4つの観点(金融経済学・計量経済学・実装・レッドチーム)で並列評価し、主要な定量的主張は Shiller の ie_data.xls を実際にダウンロードして回帰を再現する形で独立検証した。要点:

**A1. アイデア自体は王道であり、計画の Level 0 と数学的に同一。**
P = EPS × PER の分解に基づく予測は、Bogle の期待リターン分解、Grinold–Kroner 公式、Campbell–Shiller のバリュエーション回帰、GMO の7年予測、Ferreira & Santa-Clara (2011) の sum-of-the-parts(Goyal–Welch 批判を生き残った数少ない OOS 有効手法)と同じ系譜。`ln P̂ = ln P + g + λ(ln PER* − ln PER)` は「EPS成長で前進+PERを中央値へ部分回帰」そのものであり、依頼者案は λ=1(完全回帰)の特殊ケースに当たる。

**A2. 「一次関数」は対数空間でのみ成立する(検証済み)。**
生の価格・EPS水準での線形回帰は、①残差分散が価格水準の2乗でスケール(深刻な分散不均一)、②P と EPS は I(1) トレンド系列なので水準回帰は教科書的な見せかけの回帰(R² は壮大、t 値は無意味)、③係数が直近15年のデータに支配され、初期サンプルで負の価格すらフィットしうる。対数空間なら恒等式が厳密に保存され、係数は%効果として解釈できる。

**A3. 1年でのPER回帰は「約9%」しか進まない(一次データで再現検証済み)。**
- 月次 log CAPE の AR(1) ≈ 0.9957、年次換算 φ ≈ 0.90–0.93(戦後は0.93–0.95)。半減期 ~7年(戦後 ~10年)。ADF検定は単位根を棄却できない(λ=0 も排除されない)
- 1年先リターンを log CAPE ギャップに回帰: λ = 0.07–0.11(30年ローリング中央値仕様で 0.089、NW t = −2.35)、R² = 2–6%。**λ=0.2 以上を支持する仕様は存在しなかった**
- 10年ホライズンでは CAPE の R² ≈ 0.43(Vanguard, 1926–2011)に対し、1年では ~0.04。バリュエーションは長期予測の道具であり、1年予測では「分布の中心をわずかに傾け、歪みを与える」役割に留まる — Level 0 の点予測がランダムウォーク+ドリフトに見える挙動になるのは正常

**A4. 「中央値」の選択は事実ではなくモデリング上の最大の決定(検証済み)。**
全期間(1881–2026)中央値 = 16.60、post-1990 平均 = 27.2。CAPE は1991年以降、2009年初頭の数ヶ月(底13.3)を除き一貫して全期間中央値の上にある。λ=1 の下で参照値を 16.6 → 26.6 に変えると目標価格が約47対数ポイント動く。また1917–21年・1968–82年のインフレレジームでは逆方向に失敗する(「割安」がさらに10年割安になり実質価格は−50〜60%)。よって参照値はポイントインタイムのローリング窓とし、レジーム賭けであることを明示する。マクロ条件付きフェアバリュー(Vanguard の fair-value CAPE: インフレ・実質金利で条件付け)は Level 2+ の改良候補。

**A5. trailing EPS と直近成長外挿は両方とも危機時に符号ごと間違える(検証済み)。**
2007→2008 as-reported EPS −77.5%、Q4-08 −$23.25(史上初の負の四半期)、trailing P/E は2009年5月に記録の~124。TTM EPS の底は Q1-09 の $6.86。一方その後12ヶ月の指数は約+68%。2020–21年も同型(trailing P/E ~40 → +27%年)。長期実質 E10 成長は 1.6–1.9%/年(1871–2023、推定法3種とも 1.5–2% 内)で、これを g の縮小先とする。

**A6. 歴史的PERレンジをそのまま予測帯にしない。**
無条件のPERレンジ(150年で CAPE 5〜44)は1年先の条件付き分布よりはるかに広く、中心も違う。バンドは λ 倍に縮めて適用し(§5.1)、本番の確率区間は Level 4/5 の校正済み分位点が担う。構造バンドはボラティリティ急騰時(2008、2020)に広がれない等分散の帯なので、診断オーバーレイの位置づけとする。
